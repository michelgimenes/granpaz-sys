## 2. Contexto e Escopo

### Problem Statement
A aquisição de clientes para o Plano Granpaz exige uma Landing Page de alta conversão que, simultaneamente, respeite rigorosamente as diretrizes da SUSEP e do Código de Defesa do Consumidor (CDC). A ausência de um frontend público estaticamente gerado e otimizado para Core Web Vitals resulta em altas taxas de rejeição (Bounce Rate) e perda de receita. 

Sob a ótica jurídica, a utilização de termos proibidos (ex: "nosso seguro", "nós garantimos") ou a omissão do Disclaimer de Estipulante no rodapé expõe a administradora a multas milionárias e processos do PROCON por venda casada ou propaganda enganosa. Este módulo estabelece a infraestrutura de Edge Rendering (SSG/ISR) para a Landing Page, o componente de Checkout Público (Air-Gap de Seguradora) e a validação automatizada de copywriting para garantir conformidade contínua.

### Objetivos
| Categoria | Métrica / KPI | Alvo |
| :--- | :--- | :--- |
| **Performance** | Largest Contentful Paint (LCP) da Landing Page | `< 1.2s` (Mobile 4G) |
| **Performance** | Cumulative Layout Shift (CLS) | `< 0.05` |
| **Conversão** | Taxa de conversão (Visitante → Submissão de Contrato) | `> 4.5%` |
| **Compliance** | Ocorrências de termos proibidos ("nosso seguro") no DOM | `0` (Bloqueio em Build Time) |
| **SEO** | Score Lighthouse (Performance + SEO) | `> 95/100` |

### Out of Scope (Não será implementado nesta versão)
1.  **Área Logada do Cliente:** Dashboard, App Mobile e Autogestão (Cobertos por outros módulos).
2.  **Blog Institucional / CMS Complexo:** Foco estrito na Landing Page de conversão (Granpaz).
3.  **Cálculo de Prêmio de Seguro em Tempo Real:** O frontend público exibe apenas o valor do Clube/Mensalidade.
4.  **Integração com Chatbots de IA:** Suporte inicial via WhatsApp (Link estático).
5.  **Marketplace de Benefícios:** Listagem de rede credenciada (Fase 2).

### Premissas e Restrições
├── **Stack:** Next.js 14+ (App Router), React Server Components (RSC), Tailwind CSS, Vercel/Cloudflare Edge.
├── **Compliance:** SUSEP (Circular 666/2022), CDC (Art. 39 - Venda Casada), LGPD (Cookies e Rastreamento).
├── **Air-Gap Obrigatório:** O Checkout Público NUNCA carrega, exibe ou permite seleção de `seguradoras` ou `capital_segurado`.
└── **Imutabilidade de Copy:** Textos institucionais são versionados no código (i18n/JSON). Alterações exigem PR com aprovação do Jurídico.

### Glossário
| Termo | Definição |
| :--- | :--- |
| **Estipulante** | Pessoa jurídica (Saúde & Proteção) que contrata seguro coletivo. O frontend deve deixar isso explícito. |
| **Air-Gap de Seguradora** | Isolamento onde o cliente final seleciona apenas o "Plano Comercial", sem saber ou escolher a Seguradora parceira. |
| **Venda Casada** | Prática ilegal (CDC Art. 39, I) de condicionar a venda de um serviço à aquisição de outro. O seguro deve ser benefício intrínseco ou opcional claro. |
| **SSG/ISR** | Static Site Generation / Incremental Static Regeneration. Estratégia de cache para performance máxima. |
| **Disclaimer** | Aviso legal obrigatório no rodapé detalhando o papel da Estipulante e da Seguradora. |

### 3.3 Architecture Decision Records (ADRs)
| ID | Decisão | Alternativas Consideradas | Justificativa | Consequências |
| :--- | :--- | :--- | :--- | :--- |
| **ADR-01** | **SSG com ISR (Revalidação 1h)** | CSR (Create React App) ou SSR puro. | LCP < 1.2s é crítico para conversão em Mobile 4G. SSG entrega HTML pronto na Edge. ISR permite atualizar preços de planos sem novo deploy. | Exige Edge Network (Vercel/Cloudflare). |
| **ADR-02** | **Air-Gap de Seguradora no Checkout** | Exibir "Seguradora Parceira X" no checkout. | 🚨 **COMPLIANCE SUSEP:** Evita caracterização de corretagem direta. O cliente compra o "Clube Granpaz". A Seguradora é definida no Backoffice (Módulo 2). | Frontend só conhece `plano_id` e `valor_mensalidade`. |
| **ADR-03** | **Validação de Copy em Build Time** | Revisão manual periódica. | Previne que desenvolvedores ou_marketing_ injetem termos proibidos ("nosso seguro") via CMS ou código. | Script de linting customizado (`eslint-plugin-susep`) bloqueia o build. |
| **ADR-04** | **Disclaimer Hardcoded no Layout** | CMS dinâmico para o rodapé. | O rodapé legal é imutável sem aprovação jurídica. Hardcode garante que nenhuma página pública exista sem o aviso de Estipulante. | Alterações no CNPJ da Seguradora parceira exigem PR e Code Review do DPO. |

## 4. Especificação Funcional Detalhada

### 4.1 Regras de Negócio (Determinísticas)
| ID | Regra de Negócio (SE / ENTÃO / SENÃO) |
| :--- | :--- |
| **RN-01** | 🛡️ **SE** a página for renderizada no domínio público **ENTÃO** o rodapé DEVE conter o Disclaimer de Estipulante, CNPJ da Administradora e aviso SUSEP. **SENÃO** (se faltar), o build falha (Linting Error). |
| **RN-02** | 🚨 **SE** o copywriting contiver termos como "nosso seguro", "nós garantimos", "compre nosso seguro" ou "corretora" **ENTÃO** o pipeline de CI/CD bloqueia o deploy. **EXCEÇÃO:** Nenhuma. (Air-Gap Semântico). |
| **RN-03** | 💰 **SE** o usuário acessar o Checkout Público **ENTÃO** a API deve retornar APENAS `plano_id`, `nome`, `valor_base` e `tipo`. **SENÃO** (se tentar retornar `seguradora_id` ou `capital_segurado`), a API lança exceção de segurança e o frontend exibe erro genérico. |
| **RN-04** | ⚖️ **SE** o usuário tentar contratar plano `FAMILIAR` **ENTÃO** o checkout deve exibir claramente que a "Proteção Funeral é um benefício intrínseco do Clube". **SENÃO** (se for interpretado como venda casada), o PROCON pode multar. O termo "Seguro" deve ser substituído por "Proteção" ou "Benefício". |
| **RN-05** | 🔒 **SE** o usuário clicar em "Cancelar" ou "Voltar" no Checkout **ENTÃO** o sistema deve limpar o estado local (LocalStorage/SessionStorage) de dados sensíveis (CPF, Endereço). **EXCEÇÃO:** UTM parameters de marketing são preservados. |

### 4.4 Edge Cases e Exceções (30% da Seção - Foco Crítico)
| Cenário | Condição / Falha | Comportamento Esperado do Sistema |
| :--- | :--- | :--- |
| **EC-01: Falha na API de Planos** | A API `GET /planos` retorna `500` ou timeout. | O Checkout exibe estado de "Manutenção Técnica" com CTA para WhatsApp do Suporte. A LP continua no ar (SSG Cache). |
| **EC-02: Bot de Scraping / Fraude** | IP conhecido por fraude tenta submeter 100 contratos em 1 minuto. | WAF (Cloudflare/AWS WAF) bloqueia o IP com `403 Forbidden`. Rate Limiting na API (`10 req/min/IP`). |
| **EC-03: JS Desabilitado no Browser** | Usuário com NoScript tenta acessar o Checkout. | A LP (SSG) é legível. O Checkout exibe mensagem: "Habilite o JavaScript para contratar o Plano Granpaz". |
| **EC-04: Conflito de CPF (409)** | Usuário tenta contratar mas já possui contrato ativo (`AGUARDANDO_APROVACAO`). | Frontend trata o `409 Conflict` e exibe modal: "Você já possui uma proposta em análise. Acompanhe pelo App." com link para Login. |
| **EC-05: Alteração de Preço via ISR** | O preço do plano muda no DB, mas o Edge Cache ainda tem o HTML antigo. | O ISR (Incremental Static Regeneration) atualiza a página em background. O Checkout sempre busca o preço *fresco* via API no momento do clique. |
| **EC-06: Violação de Venda Casada** | Usuário reclama que foi "obrigado" a contratar o seguro. | A Copy da LP e o Checkout possuem checkbox (ou texto claro) informando que o Granpaz é um "Clube de Benefícios com Proteção Intrínseca". O termo "Seguro" é evitado na oferta principal. |

### 4.5 Validações de Entrada (Frontend & API)
| Campo | Regra | Mensagem de Erro | HTTP Code |
| :--- | :--- | :--- | :--- |
| `utm_source` | String, max 50 chars | - (Sanitizado silenciosamente) | - |
| `plano_id` | UUID válido, ativo no DB | "Plano selecionado indisponível." | 400 |
| `cpf` | Algorítmica + Unique (Módulo 1) | "CPF já possui proposta ativa." | 409 |
| `email` | RFC 5322, Lowercase | "E-mail inválido." | 400 |

## 5. Especificação Não-Funcional

### 5.1 Performance (Core Web Vitals)
├── **LCP (Largest Contentful Paint):** `< 1.2s` (Imagem Hero otimizada em WebP/AVIF, `priority={true}`).
├── **FID (First Input Delay):** `< 50ms` (Hydration seletiva de componentes interativos).
├── **CLS (Cumulative Layout Shift):** `< 0.05` (Dimensões explícitas em todas as imagens e iframes).
└── **TTFB (Time to First Byte):** `< 200ms` (Edge Caching global).

### 5.2 Escalabilidade
├── **Edge Network:** Distribuição via Vercel Edge Network ou Cloudflare Pages. Suporta picos de tráfego (ex: Black Friday) sem escalar servidores de origem.
├── **API:** Rate Limiting de `100 req/s` por IP para endpoints públicos de leitura (`/planos`).

### 5.3 Resiliência
├── **SLO:** `99.99%` de disponibilidade para a Landing Page (SSG).
├── **Fallback de API:** Se a API de Configurações cair, o Checkout usa um *hardcoded fallback* de planos básicos (apenas para não perder a venda, com posterior reconciliação manual).
├── **Cache Stale-While-Revalidate:** O Edge serve o HTML antigo enquanto busca o novo em background.

### 5.4 Segurança e Compliance
├── **CSP (Content Security Policy):** `default-src 'self'; script-src 'self' 'unsafe-inline' (apenas para GTM); img-src 'self' data: https:;`
├── **Sanitização:** Inputs do Checkout são sanitizados contra XSS antes de enviados à API.
├── **LGPD:** Banner de Cookies com opt-in explícito para rastreamento de marketing (Meta Pixel, GA4). Cookies estritamente necessários isentos.
├── **Air-Gap Semântico:** Script de CI (`eslint-plugin-susep`) varre o código fonte buscando regex de termos proibidos.

### 5.5 Observabilidade
├── **Métricas (Web Vitals API):**
│   ├── `web_vitals_lcp_seconds{page="landing"}`
│   ├── `web_vitals_cls_score{page="checkout"}`
│   └── `checkout_conversion_rate{utm_source="meta"}`
├── **Logs (Edge):** Logs de acesso da CDN (Cloudflare/Vercel) para análise de geo-localização e bloqueios de WAF.
└── **Alertas:** Queda na taxa de conversão > 20% em relação à média móvel de 7 dias.

### 5.6 Acessibilidade e i18n
├── **A11y (WCAG 2.1 AA):**
│   ├── Contraste mínimo 4.5:1 em todos os textos.
│   ├── Navegação completa via teclado (Focus Trap em Modais).
│   ├── Tags `alt` descritivas em todas as imagens.
│   ├── Hierarquia de headings (`h1` -> `h2` -> `h3`) estrita.
└── **i18n:** Preparado para PT-BR e ES (Espanhol). Moeda formatada via `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })`.

## 7. Estratégia de Entrega e Mitigação de Riscos

### 7.1 Plano de Rollback/Migração
├── **Migração:** Greenfield. Substituição da LP legada (se houver) via troca de DNS ou Rota.
├── **Rollback:** Reversão do deploy na Vercel/Cloudflare (1 clique). O SSG anterior é restaurado instantaneamente.
└── **Zero Downtime:** Deploy atômico. A nova versão só recebe tráfego após passar nos testes de Lighthouse CI.

### 7.2 Feature Flags
├── `NEW_CHECKOUT_FLOW_ENABLED`: Se `true`, habilita o novo fluxo de checkout multi-step. Se `false`, redireciona para LP legada ou formulário simples.
├── `MARKETING_PIXELS_ENABLED`: Controla o disparo de Meta Pixel / GA4 (desligado em caso de problema de LGPD).

### 7.3 Matriz de Riscos
| Risco | Probabilidade | Impacto | Plano de Mitigação |
| :--- | :--- | :--- | :--- |
| **Multa SUSEP/PROCON por Copy** | Baixa | Crítico | 🛡️ **Air-Gap Semântico.** Script de CI bloqueia termos proibidos. Rodapé com Disclaimer é hardcoded e imutável sem PR do Jurídico. |
| **Baixa Conversão (LCP Alto)** | Média | Alto (Receita) | Lighthouse CI no pipeline. Bloqueia deploy se LCP > 1.5s. Imagens otimizadas via Next/Image. |
| **Venda Casada (CDC Art. 39)** | Média | Crítico | Copy revisada por jurídico. Checkbox de ciência de que é um "Clube de Benefícios". |
| **Fraude de Leads (Bots)** | Alta | Médio | WAF na Edge. Rate Limiting na API. Captcha invisível (reCAPTCHA v3) no submit. |

### 7.4 Dependências Externas
├── **Módulo 1 (Cadastro):** O Checkout Público consome a API de submissão de contratos do Módulo 1.
├── **Jurídico/DPO:** Aprovação obrigatória de qualquer alteração no `copy.json` ou no Disclaimer do Rodapé.
├── **Marketing:** Fornecimento de UTMs e configuração de Pixels de conversão.
