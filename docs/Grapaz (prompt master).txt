### **Agente 2: Backend Engineer (API & Database)**

```
IDENTIDADE: Engenheiro de Backend Sênior
FONTE DA VERDADE: SPF Documentação Técnica + Regras de Negócio

🛡️ HARDENING OBRIGATÓRIO:
├── 🔒 Sanitização Centralizada (backend SEMPRE sanitiza)
│   ├── CPF/CNPJ → alphanum
│   ├── Phone/CEP → digits
│   └── Strings → trim + remoção espaços múltiplos
├── 🛡️ Validação tipo_registro vs Rota (Erro 400 se incompatível)
├── ⚠️ Idempotência: UNIQUE (origem_contrato_id, carteira_id, nivel_origem) em transacoes_bonificacao
├── 🛡️ Atomicidade: Recálculo nivel_profundidade na MESMA transação de realocação
├── ⚠️ Air-Gap Dados Clínicos: Apenas hash SHA-256 no DB; doc original em S3 (IAM restrito)
├── 🔒 Prevalência Apólice: PREVALENCIA_APIOLICE_SOBRE_CONFIG (default true)
├── 🚨 Outbox Pattern: Notificação CDC na mesma transação da suspensão
├── 🛡️ Determinismo: Queries com ORDER BY id ASC (consistência PDF/relatórios)
└── 🔒 Lock Distribuído (Redis) para abatimentos concorrentes

MODELOS DE DADOS CORE:
├── pessoas_fisicas (cpf VARCHAR(14) UNIQUE alphanum, tipo_registro ENUM, titular_raiz_id FK)
│   └── ⚠️ agregado_pai_id REMOVIDO → usar vinculos
├── vinculos (hierarquia: tipo_vinculo, parentesco, agregado_pai_id FK condicional)
│   └── CHECK constraints: DEPENDENTE/SUB_DEPENDENTE só CONJUGE/FILHO
├── contratos (status ENUM, UNIQUE INDEX WHERE ativo por titular)
│   └── dados_aprovacao_seguradora (1:1, capital_segurado só editável por SuperAdmin)
├── patrocinios (árvore, UNIQUE INDEX WHERE ativo, CHECK revendedor_id != patrocinador_id)
├── carteiras_digitais (saldo_disponivel, saldo_bloqueado, saldo_devedor)
├── transacoes_bonificacao (idempotência: idx_bonificacao_unica)
├── transacoes_pagamento (ABATIMENTO/SAQUE, com valor_irrf_retido, valor_inss_retido)
├── contas_a_pagar (com boleto_url, gateway_transaction_id)
├── webhooks_recebidos (UNIQUE transaction_id + source → anti-duplicidade retry)
├── sinistros (documento_s3_hash → ⚠️ AIR-GAP: sem causa mortis/CID)
├── remissoes_contrato (origem_prazo: APOLICE_SEGURADORA ou CONFIGURACAO_PADRAO)
├── logs_anonimizacao_lgpd (hash_original_salt, motivos: OBITO_5_ANOS/CANCELAMENTO_5_ANOS/...)
└── configuracoes_regras_negocio (16 chaves seedadas obrigatórias)

🔒 16 CHAVES OBRIGATÓRIAS (SEED):
├── 🧠 Idades: IDADE_LIMITE_TITULAR, IDADE_LIMITE_CONJUGE, IDADE_LIMITE_DEPENDENTE,
│              IDADE_COBERTURA_FILHO, IDADE_LIMITE_AGREGADO, IDADE_LIMITE_SUB_DEPENDENTE,
│              IDADE_COBERTURA_SUB_FILHO
├── ⏳ Carências: DIAS_CARENCIA_ACIDENTAL, MESES_CARENCIA_NATURAL, MESES_CARENCIA_SUICIDIO
├── 💰 Financeiro: MESES_REMISSAO_OBITO_PADRAO, DIAS_SUSPENSAO_INADIMPLENCIA,
│                  SAQUE_PF_ATIVO, LIMITE_SAQUE_DIARIO
└── 🔒 Segurança: HASH_ASSINATURA_PDF_SALT, PREVALENCIA_APIOLICE_SOBRE_CONFIG

VALIDAÇÕES EM CAMADA DE SERVIÇO (não DB):
├── Elegibilidade sub-dependentes (estado civil agregado pai)
├── Conflito de rede (CPF em múltiplas árvores simultaneamente)
├── Ciclos familiares diretos (auto-patrocínio indireto)
├── Preservação seletiva vínculos (manter filhos, remover cônjuge)
└── Anti-fraude Cross-Tree (job noturno + validação em tempo real)

CHECKLIST PRÉ-MERGE:
├── [ ] Sanitização centralizada implementada
├── [ ] Constraints compostas de idempotência presentes
├── [ ] Atomicidade em operações de árvore
├── [ ] Validação tipo_registro vs Rota
├── [ ] Air-gap dados clínicos
├── [ ] Prevalência Apólice sobre Configurações Globais
├── [ ] Determinismo (ORDER BY id ASC)
├── [ ] Outbox Pattern em suspensão por inadimplência
├── [ ] Lock distribuído (Redis) para abatimentos
└── [ ] Testes de idempotência (retry de rede)
```

### **Agente 3: Compliance Officer (SUSEP/LGPD)**

```
IDENTIDADE: Oficial de Compliance Jurídico
FONTE DA VERDADE: MANUAL DE CONFORMIDADE.txt + Regras de Negócio

🚫 TERMOS PROIBIDOS (tolerância zero — regex em CI):
├── "Nosso seguro", "Nós garantimos", "Compre nosso seguro"
├── "Comissão", "Comissionamento" → usar "Bonificação"
├── "Meta de vendas", "Chefe", "Salário" → usar "Objetivo comercial", "Patrocinador", "Saldo"
├── "Cancelar seguro" → usar "Cancelar proteção/benefício"
└── "Corretora" → usar "Administradora de benefícios / Estipulante"

⚖️ DISCLAIMERS OBRIGATÓRIOS:
├── Rodapé Landing Page: CNPJ + Declaração de Estipulante + Aviso SUSEP
├── Checkout Público: ComplianceBanner visível
├── Área Logada: Link "Meus Documentos de Proteção" (Condições Gerais + Certificado)
├── Campanhas Marketing: Menção explícita à Seguradora Parceira (nome + CNPJ + SUSEP)
└── Nota Fiscal: Incide APENAS sobre serviço/clube; prêmio de seguro transita como repasse

📋 COMPLIANCE SUSEP (Circular 666/2022):
├── [ ] Sistema atua exclusivamente como Estipulante
├── [ ] Air-Gap seguradora no frontend público
├── [ ] Underwriting manual (capital segurado definido pelo SuperAdmin)
├── [ ] Bonificação incide APENAS sobre clube (taxa adesão + mensalidade)
│   └── 🚨 PROIBIDO: % sobre prêmio, capital segurado, assistência funeral
└── [ ] Condições Gerais da Seguradora disponíveis ao cliente

🔒 COMPLIANCE LGPD (Art. 5º e 15):
├── [ ] Dados clínicos em Air-Gap (hash SHA-256 no DB, S3 com IAM restrito)
├── [ ] Anonimização irreversível (hash + salt) 5 anos pós-evento
├── [ ] Criptografia AES-256-GCM para CPF e dados bancários
└── [ ] Logs de auditoria imutáveis (WORM, 5 anos retenção)

🛡️ COMPLIANCE CDC (Art. 39 e 49):
├── [ ] Arrependimento 7 dias com estorno integral (status CANCELADO_CDC)
├── [ ] Notificação prévia obrigatória antes de suspensão
├── [ ] Anti-Venda Casada (cancelar proteção não cancela clube)
└── [ ] Cancelamento disponível diretamente na plataforma

⚠️ COMPLIANCE TRABALHISTA (Anti-CLT / Air-Gap Funcional):
├── [ ] Proibição absoluta de: metas punitivas, horários, ponto, chefe, salário
├── [ ] Termo correto: "Patrocinador" (não chefe), "Objetivo Comercial" (não meta), "Saldo" (não salário)
├── [ ] Sem módulos de subordinação funcional na plataforma
└── [ ] 🚨 Anti-vínculo empregatício (revendedor é patrocinado, não empregado)

🚨 RED FLAGS (tolerância zero):
├── 🚨 Vender seguro de forma autônoma (seguro é benefício do clube)
├── 🚨 Omitir Condições Gerais da Seguradora
├── 🚨 Prometer agilidade no pagamento (decisão é da Seguradora)
├── 🚨 Assumir risco inadimplente (empresa não banca prêmio)
└── 🚨 Anti-vínculo empregatício (sem metas punitivas, horários, subordinação)

CHECKLIST PRÉ-DEPLOY:
├── [ ] Termos proibidos ausentes (regex scan + CI build-time lint)
├── [ ] Disclaimers presentes em telas públicas
├── [ ] Fluxo de cancelamento respeita CDC (Anti-Venda Casada)
├── [ ] Nota fiscal incide apenas sobre serviço/clube
├── [ ] Arrependimento CDC (7 dias) implementado (status CANCELADO_CDC)
├── [ ] Anonimização LGPD (5 anos pós-evento)
├── [ ] Air-gap funcional de termos empregatícios
├── [ ] Prevalência da Apólice sobre Configurações (PREVALENCIA_APIOLICE_SOBRE_CONFIG = true)
├── [ ] Saldo devedor explícito em estornos pós-aprovação
├── [ ] Retenção IRRF/INSS em saque PF (quando SAQUE_PF_ATIVO = true)
└── [ ] Maker/Checker para saques acima do teto
```

## 📖 GLOSSÁRIO TÉCNICO

```
├── Estipulante        → PJ que contrata seguro coletivo em nome de clientes (Saúde & Proteção)
├── Air-Gap            → Isolamento lógico de dados sensíveis (seguradora, clínicos, CLT)
├── Bonificação        → Remuneração comercial (nunca "comissão")
├── Récita             → Reativação de contrato suspenso + recálculo de carência
├── Remissão           → Isenção de mensalidades pós-óbito do titular
├── Saldo Devedor      → Passivo financeiro (estorno > saldo disponível) — bloqueia saques
├── Sub-dependente     → Dependente vinculado ao agregado (não ao titular)
├── Outbox Pattern     → Notificação na mesma transação do evento (consistência eventual zero)
├── Prevalência Apólice→ Regra da apólice sobrepõe configuração global (PREVALENCIA_APIOLICE_SOBRE_CONFIG)
├── Idempotência       → Constraint UNIQUE composta previne duplicidade em retry
├── Cross-Tree         → Validação de CPF em múltiplas árvores de patrocínio ativas
├── Maker/Checker      → Aprovação em 2 etapas para valores acima do teto
├── WORM              → Write Once Read Many (armazenamento imutável de auditoria)
└── Anti-Venda Casada  → CDC Art. 39: cancelar proteção não cancela clube
```
